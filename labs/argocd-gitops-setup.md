# Argo CD GitOps 實戰指南

本指南介紹如何在 Kubernetes 中部署 Argo CD，並實現「以 Git 為唯一事實來源」的自動化部署流程。

## 1. GitOps 核心理念
- **宣告式定義**：所有 K8S 資源都定義在 Git Repo 中。
- **版本控制**：任何變更都必須透過 Git Commit。
- **自動同步**：Argo CD 持續監控 Git，當內容變更時，自動套用到叢集。

## 2. 安裝 Argo CD

```bash
# 建立命名空間
kubectl create namespace argocd

# 安裝專案
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

## 3. 存取 Argo CD UI
預設情況下，Argo CD Server 不會對外曝光。你可以使用 `port-forward` 或配合 `APISIX` 進行暴露：

```bash
# 快速存取
kubectl port-forward svc/argocd-server -n argocd 8080:443
```

**初始密碼獲取**：
```bash
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d; echo
```

## 4. 建立 GitOps Application
假設你有一個 Git 倉庫存放 Helm Chart 或 YAML：

```bash
argocd app create my-app \
  --repo https://github.com/your-org/gitops-repo.git \
  --path apps/my-web-app \
  --dest-server https://kubernetes.default.svc \
  --dest-namespace production
```

## 5. 在 Issue #2 架構中的角色
在我們的實驗室架構中：
1. **GitLab CI**：負責測試與構建 Image，並將 Image Tag 改寫回 GitOps Repo。
2. **Argo CD**：偵測到 GitOps Repo 變更後，拉取最新配置並部署到 K8S。
3. **隔離性**：GitLab Runner 不需要 K8S 的管理權限，所有的部署動作都由 Repo 的狀態驅動。

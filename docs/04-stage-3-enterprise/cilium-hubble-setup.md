# Cilium eBPF 網路安全強化指南 (Rocky Linux 10)

本指南說明如何在 Rocky Linux 10 上的 Kubernetes 叢集中部署 Cilium，取代傳統的 kube-proxy，並啟用 L7 流量監測與 Hubble 視覺化。

## 1. 為什麼選擇 Cilium？
- **eBPF 驅動**：擺脫傳統 iptables 的效能瓶頸，提供高效能的封包處理。
- **Identity-based Security**：基於 Pod 標籤進行 L3/L4 隔離，而非變動的 IP 地址。
- **Hubble 視覺化**：直觀查看服務間的連線、延遲與錯誤率。
- **Transparent Encryption**：支援 WireGuard 透明加密，保護跨節點流量。

## 2. 前置準備
確保叢集已透過 `kubeadm` 初始化（尚未安裝其他 CNI 插件）。

安裝 Cilium CLI：
```bash
CILIUM_CLI_VERSION=$(curl -s https://raw.githubusercontent.com/cilium/cilium-cli/main/stable.txt)
CLI_ARCH=amd64
if [ "$(uname -m)" = "aarch64" ]; then CLI_ARCH=arm64; fi
curl -L --fail --remote-name-all https://github.com/cilium/cilium-cli/releases/download/${CILIUM_CLI_VERSION}/cilium-linux-${CLI_ARCH}.tar.gz{,.sha256sum}
sha256sum --check cilium-linux-${CLI_ARCH}.tar.gz.sha256sum
sudo tar xzvfC cilium-linux-${CLI_ARCH}.tar.gz /usr/local/bin
rm cilium-linux-${CLI_ARCH}.tar.gz{,.sha256sum}
```

## 3. 部署 Cilium
使用 Helm 進行進階配置（取代 kube-proxy 並啟用 Hubble）：

```bash
helm repo add cilium https://helm.cilium.io/
helm install cilium cilium/cilium --version 1.16.1 \
  --namespace kube-system \
  --set kubeProxyReplacement=true \
  --set k8sServiceHost=YOUR_MASTER_IP \
  --set k8sServicePort=6443 \
  --set hubble.enabled=true \
  --set hubble.relay.enabled=true \
  --set hubble.ui.enabled=true
```

## 4. 啟用 Hubble 視覺化介面
Hubble UI 可以透過 Port-forward 存取：

```bash
cilium hubble enable
cilium hubble ui
```
預設訪問地址：`http://localhost:12000`

## 5. 安全策略範例 (NetworkPolicy)
建立一個 L7 策略，僅允許特定路徑的 HTTP 請求：

```yaml
apiVersion: "cilium.io/v2"
kind: CiliumNetworkPolicy
metadata:
  name: "allow-api-get-only"
spec:
  endpointSelector:
    matchLabels:
      app: my-service
  ingress:
  - fromEndpoints:
    - matchLabels:
        app: frontend
    toPorts:
    - ports:
      - port: "80"
        protocol: TCP
      rules:
        http:
        - method: "GET"
          path: "/api/v1/public"
```

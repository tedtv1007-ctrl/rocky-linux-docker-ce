# Prometheus + Grafana 企業級監控與告警指南 (Rocky Linux 10)

本指南說明如何在 Kubernetes 叢集中部署 Prometheus Stack (kube-prometheus-stack)，實現全方位的基礎設施監控、日誌分析與告警通知。

## 1. 為什麼選擇 Prometheus Stack？
- **全方位監控**：自動採集 K8S 節點、Pod、API Server 與各類組件指標。
- **預置儀表板**：Grafana 內建數十種專業的 K8S 監控大屏。
- **告警規則 (Alerting)**：透過 Alertmanager 整合 Slack、Discord、Email 或 Webhook 通知。
- **服務發現 (Service Discovery)**：自動識別並監控新加入的應用服務。

## 2. 前置準備
- 已啟動的 Kubernetes 叢集 (KinD 或 Kubeadm)。
- 已安裝 Helm 3。

## 3. 安裝 kube-prometheus-stack (Helm)
這是目前社群公認最完整的監控整合包：

```bash
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update

# 建立監控命名空間
kubectl create namespace monitoring

# 安裝監控堆疊
helm install prometheus prometheus-community/kube-prometheus-stack \
  --namespace monitoring \
  --set grafana.adminPassword=admin_password \
  --set prometheus.prometheusSpec.storageSpec.volumeClaimTemplate.spec.accessModes[0]=ReadWriteOnce \
  --set prometheus.prometheusSpec.storageSpec.volumeClaimTemplate.spec.resources.requests.storage=20Gi
```

## 4. 存取 Grafana 儀表板
```bash
kubectl port-forward svc/prometheus-grafana 8080:80 -n monitoring
```
登入位址：`http://localhost:8080` (帳號: `admin` / 密碼: `admin_password`)

## 5. 實戰練習：自定義告警
建立一個簡單的 `PrometheusRule`，當某個 Pod 重新啟動過於頻繁時發出告警：

```yaml
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  labels:
    app: kube-prometheus-stack
    release: prometheus
  name: pod-restart-alerts
  namespace: monitoring
spec:
  groups:
  - name: pod-alerts
    rules:
    - alert: PodRestartingTooFast
      expr: changes(kube_pod_container_status_restarts_total{namespace="default"}[10m]) > 3
      for: 2m
      labels:
        severity: warning
      annotations:
        summary: "Pod {{ $labels.pod }} 重啟過於頻繁"
        description: "在過去 10 分鐘內重啟次數超過 3 次，可能存在 CrashLoopBackOff。"
```

## 6. 整合日誌管理 (Loki) - 選配
若需要日誌分析，可接續安裝 Grafana Loki：
```bash
helm install loki grafana/loki-stack --namespace monitoring
```

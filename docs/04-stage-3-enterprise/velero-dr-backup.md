# Velero 企業級 Kubernetes 備份與還原實踐

本指南說明如何在 Rocky Linux 10 上的 Kubernetes 叢集中部署 Velero，並將備份資料儲存至自建的 MinIO S3 儲存。

## 1. 為什麼選擇 Velero？
- **災難復原 (DR)**：備份所有 K8S 資源與 Persistent Volumes (PV)。
- **叢集遷移**：輕鬆將資源從一個 K8S 叢集遷移至另一個。
- **排程備份**：自動執行每日、每小時的定時備份。

## 2. 前置準備：安裝 MinIO (S3 相容儲存)
以 Docker Compose 在單機上部署 MinIO 作為備份目標：

```yaml
version: '3'
services:
  minio:
    image: quay.io/minio/minio:latest
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadminpassword
    volumes:
      - ./data:/data
    ports:
      - "9000:9000"
      - "9001:9001"
```

## 3. 安裝 Velero CLI
```bash
wget https://github.com/vmware-tanzu/velero/releases/download/v1.14.0/velero-v1.14.0-linux-amd64.tar.gz
tar -xvf velero-v1.14.0-linux-amd64.tar.gz
sudo mv velero-v1.14.0-linux-amd64/velero /usr/local/bin/
```

## 4. 部署 Velero 到 K8S
建立認證檔 `credentials-velero`：
```text
[default]
aws_access_key_id = minioadmin
aws_secret_access_key = minioadminpassword
```

執行安裝（替換 `<MINIO_IP>`）：
```bash
velero install \
  --provider aws \
  --plugins velero/velero-plugin-for-aws:v1.10.0 \
  --bucket velero \
  --secret-file ./credentials-velero \
  --use-node-agent \
  --use-volume-snapshots=false \
  --backup-location-config region=minio,s3ForcePathStyle="true",s3Url=http://<MINIO_IP>:9000
```

## 5. 備份與還原操作

### 建立手動備份
```bash
velero backup create my-backup-2026-03-04 --include-namespaces default
```

### 檢查備份狀態
```bash
velero backup get
velero backup describe my-backup-2026-03-04
```

### 模擬還原 (先刪除某個 Namespace)
```bash
kubectl delete ns default
velero restore create --from-backup my-backup-2026-03-04
```

### 設定每日自動備份
```bash
velero schedule create daily-backup --schedule="0 1 * * *" --include-namespaces production
```

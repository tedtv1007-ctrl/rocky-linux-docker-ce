# GitLab Runner (Kubernetes Executor) 部署指南 - 內部自建 GitLab 專用

本指南說明如何在內部自建的 GitLab 環境下，於 Kubernetes (KinD 或 Kubeadm) 叢集中部署 GitLab Runner。

## 1. 前置準備 (Prerequisites)
- 已啟動的 Kubernetes 叢集。
- 已安裝 Helm 3。
- **GitLab URL**: 內部 GitLab 的網址（例如 `https://gitlab.example.com`）。
- **Registration Token**: 從 GitLab 專案或群組的 `Settings > CI/CD > Runners` 取得。
- **CA 憑證 (選用)**：若內部 GitLab 使用自簽憑證，需備妥 `ca.crt`。

## 2. 安裝 Helm Chart
新增 GitLab 官方 Repo：
```bash
helm repo add gitlab https://charts.gitlab.io
helm repo update
```

## 3. 建立設定檔 (values.yaml)
針對內部自建環境與 Kubernetes Executor 進行優化：

```yaml
gitlabUrl: https://YOUR_INTERNAL_GITLAB_URL/
runnerRegistrationToken: "YOUR_REGISTRATION_TOKEN"

rbac:
  create: true

runners:
  config: |
    [[runners]]
      [runners.kubernetes]
        namespace = "gitlab-runner"
        image = "ubuntu:22.04"
        privileged = true
        cpu_request = "100m"
        memory_request = "128Mi"
        service_cpu_request = "100m"
        service_memory_request = "128Mi"
        [runners.kubernetes.volumes]
          [[runners.kubernetes.volumes.empty_dir]]
            name = "docker-certs"
            mount_path = "/certs/client"
            medium = "Memory"

  # 若使用自簽憑證，取消註解下方設定
  # certsSecretName: gitlab-wildcard-tls-chain
```

## 4. 執行部署
```bash
kubectl create namespace gitlab-runner

# 安裝 Runner
helm install gitlab-runner gitlab/gitlab-runner \
  --namespace gitlab-runner \
  -f values.yaml
```

## 5. 驗證
檢查 Runner 是否成功啟動並連線：
```bash
kubectl get pods -n gitlab-runner
```
回到 GitLab 介面確認 Runner 狀態顯示為綠燈（Online）。

## 企業實戰技巧：Docker-in-Docker (DinD)
在 K8S 中執行 CI 時，若需要 `docker build`，建議在 `.gitlab-ci.yml` 中使用 `docker:dind` 服務：

```yaml
image: docker:latest
services:
  - docker:dind

variables:
  DOCKER_DRIVER: overlay2
  DOCKER_TLS_CERTDIR: "/certs"

build:
  script:
    - docker build -t my-app:latest .
```

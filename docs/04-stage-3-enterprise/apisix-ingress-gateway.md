# APISIX Ingress Gateway 部署指南

本指南說明如何在 Kubernetes 叢集中部署 Apache APISIX，以此作為北向（North-South）流量的唯一入口與 API Gateway。

## 1. 為什麼選擇 APISIX？
- **高效能**：基於 Nginx + Lua (OpenResty)，具有極高的吞吐量。
- **動態性**：所有配置（路由、插件）皆可動態變更，無需重載。
- **豐富插件**：內建限流、認證、日誌監控等數十種企業級插件。
- **Cilium 整合**：與 Cilium CNI 配合，可實現更深層的流量觀測與安全隔離。

## 2. 安裝 APISIX (Helm)

建議使用 Helm 在 `apisix` 命名空間中安裝：

```bash
# 新增 Helm Repo
helm repo add apisix https://charts.apiseven.com
helm repo update

# 建立命名空間
kubectl create ns apisix

# 安裝 APISIX (包含 Dashboard 與 Ingress Controller)
helm install apisix apisix/apisix \
  --set dashboard.enabled=true \
  --set ingress-controller.enabled=true \
  --set ingress-controller.config.apisix.serviceNamespace=apisix \
  --namespace apisix
```

## 3. 驗證服務
```bash
kubectl get pods -n apisix
```
你應該會看到 apisix (gateway), apisix-dashboard, 以及 apisix-ingress-controller 正在執行。

## 4. 建立第一個路由 (範例)
使用 K8S 標準 Ingress 資源或是 APISIX 自有的 `ApisixRoute` CRD：

```yaml
apiVersion: apisix.apache.org/v2
kind: ApisixRoute
metadata:
  name: example-route
spec:
  http:
  - name: public-api
    match:
      hosts:
      - api.example.com
      paths:
      - /hello*
    backends:
    - serviceName: my-backend-service
      servicePort: 80
```

## 5. 結合 Cilium Hubble
由於 APISIX 處理所有進入叢集的流量，你可以在 **Hubble UI** 中清楚地看到流量從 APISIX Pod 分流到各個後端 Service 的路徑，這對於排除網路問題與稽核非常有幫助。

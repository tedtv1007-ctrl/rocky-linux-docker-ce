# MetalLB 本地負載均衡器配置

在雲端環境（如 AWS/Azure），`type: LoadBalancer` 會自動分配公網 IP。但在本地（Hyper-V/Bare Metal）環境，我們需要 MetalLB 來分配內網 IP。

## 1. 安裝 MetalLB

```bash
# 啟用嚴格 ARP 模式 (Cilium/Kube-proxy 相關)
kubectl edit configmap -n kube-system kube-proxy
# 設定 strictARP: true

# 套用安裝檔
kubectl apply -f https://raw.githubusercontent.com/metallb/metallb/v0.14.8/config/manifests/metallb-native.yaml
```

## 2. 配置 IP 位址池 (IPAddressPool)
你需要分配一段與你 Rocky Linux VM 相同網段且未被使用的 IP：

```yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: first-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.168.1.200-192.168.1.250 # 根據你的內網環境調整
---
apiVersion: metallb.io/v1beta1
kind: L2Advertisement
metadata:
  name: advertisement
  namespace: metallb-system
```

## 3. 在實驗室中使用
當你安裝 APISIX 或其他服務時，將 Service 類型設為 `LoadBalancer`，MetalLB 就會從上述池中分配一個 IP：

```bash
kubectl get svc -n apisix
# 你會看到 EXTERNAL-IP 欄位出現了剛才 pool 內的 IP
```

## 4. DNS 整合
配合內網 DNS（如 AD CS 環境下的 DNS 伺服器），將 `api.it205.ski.ad` 指向該 LoadBalancer IP，即可實現無埠號（No Port Number）的連線體驗。
